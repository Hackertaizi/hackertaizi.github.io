设计模式

设计模式分类

  • 创建型模式（ 如何创建对象，让对象的创建和使用分离）
    • 单例模式、工厂模式、抽象工厂模式、建造者模式、原型模式
  • 结构型模式（ 将对象和类组合成一定的结构）
    • 适配器模式、桥接模式、装饰模式、组合模式、外观模式、享元模式、代理模式
  • 行为型模式（对象间如何相互协作，完成单个对象无法完成的任务）
    • 模板方法模式、命令模式、迭代器模式、观察者模式、中介者模式、备忘录模式、解释器模式、状态模式、策略模式、职责链模式、访问者模式

OOP（面向对象）七大原则

  • 开闭原则：对扩展开放，对修改关闭。

当应用需求发生变化时，能够尽量在原有代码上进行扩展，而不修改原有代码。

  • 里氏替换原则：继承必须保证父类所拥有的性质在子类中仍然成立。

子类尽量在继承父类方法的基础上，添加自己的方法来实现新的功能，尽量不重写父类已有的方法。

  • 依赖倒置原则：要面向接口编程，不要面向实现编程。

集合框架

一、集合框架图

解析

从上面的集合框架图中可以看出，语法形式上来看，集合框架主要有三种形式，首先是接口（图中的原点虚线框），最主要的集合接口是Collection，其次有List、Set、Map、Queue等，然后是抽象类（短横线虚线框），抽象类部分实现了接口中的方法，最后是具体的实现类(实线框），如ArrayList、LinkedList、HashSet、TreeSet、HashMap、TreeMap等。此外，还有一些集合算法，即实现集合接口的对象中的一些方法，如搜索、排序等方法。

二、Set和List的区别

    1. Set 接口实例存储的是无序的，不重复的数据。List 接口实例存储的是有序的，可以重复的元素。
    1. Set检索效率低下，删除和插入效率高，插入和删除不会引起元素位置改变 **<实现类有HashSet,TreeSet>**。
    1. List和数组类似，可以动态增长，根据实际存储的数据的长度自动增长List的长度。查找元素效率高，插入删除效率低，因为会引起其他元素位置改变 <实现类有ArrayList,LinkedList,Vector> 。

3.3 HashMap

HashMap 是一个散列表，它存储的内容是键值对(key-value)映射。

HashMap 实现了 Map 接口，根据键的 HashCode 值存储数据，具有很快的访问速度，最多允许一条记录的键为 null，不支持线程同步。

HashMap 是无序的，即不会记录插入的顺序。

HashMap 继承于AbstractMap，实现了 Map、Cloneable、java.io.Serializable 接口。

面向对象

类和对象

类是对象的抽象，类是对象的模板，对象是类的示例，类实例化后就是对象。

创建和初始化对象

创建对象使用new关键字，语法为：类名 对象名 = new 类名();

使用new关键字创建的时候，处理分配内存空间之外，还会给创建好的对象进行默认的初始化以及对类中构造器的调用。

构造器

特点

类中的构造器也称为构造方法，是在进行创建对象的时候必须要调用的。构造器有如下两个特点：

  • 1 . 必须和类的名字相同
  • 2 . 没有返回类型，也不写void

作用

  • 使用new关键字，本质上是在调用构造器。
  • 可以在构造器中初始化对象的值。

注意

一旦定义了有参数的构造器，如果要使用无参数的构造器，就必须显示定义无参数的构造器。有参数的构造器是对无参构造器的重载。

快捷键

Alt+insert快速创建构造器。

面向对象的特点

封装

使用private标识符可以将类中的属性和方法设置为私有类，从而不能被直接调用，可以定义某些方法来作为调用私有属性和类的接口。

  • Java中只有单继承，没有多继承。
  • 继承是类和类之间的一种关系。除此之外，类和类之间的关系还有依赖、组合、聚合等。
  • 子类继承父类，也就继承了父类中的所有公有属性和方法。

super方法

  • super调用父类的构造方法，必须在子类构造方法的第一行

方法重写

重写的意义

当某些父类方法的功能不能满足子类的需要时，子类需要对父类的方法进行重写

重写规则

  • 重写是建立在有几成关系的两个类间的，子类可以重写父类的方法。

  • 要重写的方法是非静态方法，静态方法不会被重写成功

  • 要重写的方法的方法名和参数列表必须和父类的相同

  • 修饰符：范围可以扩大不可以缩小(public>protected>default>private)

  • 抛出的异常：范围可以缩小不能扩大

多态

概念

同一方法可以根据发送对象的不同而采用多种不同的行为方式。多态是方法的特性，属性没有多态性。

抽象类

  • Java使用abstract关键字来声明抽象类或者抽象方法

  • 抽象类中可以没有抽象方法，但是有抽象方法的类一定要声明为抽象类。

  • 抽象类不能使用new关键字来实例化为对象，只能被继承；

  • 抽象类处理不能实例化对象之外，其他功能依然存在，属性、方法和构造方法的访问方式和普通类一样；

  • 抽象方法只有方法的声明，没有方法的实现，它是用来让继承的子类来实现的。

  • 子类继承抽象类，就必须要实现抽象类没有实现的抽象方法，否则该子类也要声明为抽象类。

  • 一个类只能继承一个抽象类，但可以实现多个接口；

泛型

一、泛型介绍

Java 泛型（generics）是 JDK 5 中引入的一个新特性, 泛型提供了编译时类型安全检测机制，该机制允许程序员在编译时检测到非法的类型。

泛型的本质是参数化类型，也就是说所操作的数据类型被指定为一个参数。

泛型可以理解为“广泛的类型”，也就是说定义时类型是未定的，需要使用时实际指定。

二、泛型方法

声明泛型方法时，类型参数声明部分（尖括号部分）要放在方法的返回类型声明之前，类型参数声明部分（尖括号部分）可以包括一个或者多个参数，中间用逗号分隔，一个泛型参数，也被称为一个类型变量，是用于指定一个泛型类型名称的标识符。

类型参数能被用来声明返回值类型，并且能作为泛型方法得到的实际参数类型的占位符。

泛型方法体的声明和其他方法一样。注意类型参数只能代表引用型类型，不能是原始类型（像 int、double、char 等）。

java 中泛型标记符：

  • E - Element (在集合中使用，因为集合中存放的是元素)
  • T - Type（Java 类）
  • K - Key（键）
  • V - Value（值）
  • N - Number（数值类型）
  • ？ - 表示不确定的 java 类型

当需要限制那些被允许传递到一个类型参数的类型种类范围时，例如，一个操作数字的方法可能只希望接受Number或者Number子类的实例。就会用到有界类型参数。

四、泛型类

泛型类的声明和非泛型类的声明类似，除了在类名后面添加了类型参数声明部分。

和泛型方法一样，泛型类的类型参数声明部分也包含一个或多个类型参数，参数间用逗号隔开。一个泛型参数，也被称为一个类型变量，是用于指定一个泛型类型名称的标识符。因为他们接受一个或多个参数，这些类被称为参数化的类或参数化的类型。

五、类型通配符

类型通配符一般是使用 ? 代替具体的类型参数。例如 List<?> 在逻辑上是 List<String>,List<Integer> 等所有 List<具体类型实参> 的父类。

类型通配符的上下界

类型通配符上限通过形如**List<? extends Number>**来定义，如此定义就是通配符泛型值接受Number及其下层子类类型。

类型通配符下限通过形如 List<? super Number> 来定义，表示类型只能接受 Number 及其上层父类类型，如 Object 类型的实例
