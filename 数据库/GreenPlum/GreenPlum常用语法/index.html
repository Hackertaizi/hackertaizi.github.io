1. 表

1.1 建表

  • table_name: 表名
  • column1、2、3：列名
  • datatype: 列数据类型（常用的有int、varchar(255)、text、timestamp、array等类型）
  • [constraint] 是可选的，用于定义列或表级别的约束（例如 NOT NULL, UNIQUE, PRIMARY KEY, FOREIGN KEY 等）。
  • DISTRIBUTED BY：指定一列或几列为分布键，数据将依照分布键分发到各个计算节点上，避免数据倾斜。
  • PARTITION BY LIST：指定一列作为分区键，分区表建表时指定的分区为初始分区。这里是通过字段建立分区，还有其他建立分区的方式，如范围分区等等。

4. 索引

4.0 命名

greenplum的索引名称是唯一的，删除索引时只会根据索引名字进行删除，不关心具体的表，所以索引命名时不能重复。

建议命名规范：idx_表名_索引字段名(有多个字段就用下划线分隔)

4.1 添加索引

  • UNIQUE：如果指定，索引将包含唯一值。
  • CONCURRENTLY：允许在不锁定表的情况下创建索引。但是，在创建索引期间，对表的写操作可能会慢一些。
  • index_name：索引的名称。
  • table_name：要在其上创建索引的表的名称。
  • method：索引方法（如B-Tree、Bitmap等）。如果不指定，将使用B-Tree。
  • column_name：要在其中创建索引的列的名称。
  • op_class：操作符类（可选）。对于某些索引方法，这允许你指定索引中使用的操作符集合。
  • SORTBY {ASC|DESC}：指定索引中列的排序顺序（仅对特定索引方法有效，如B-Tree）。
  • WITH (storage_parameter = value [, ...] )：指定索引的存储参数（可选）。

4.2 删除索引

  • IF EXISTS：这是一个可选的子句，用于在索引不存在时防止产生错误。如果指定了IF EXISTS并且索引不存在，那么命令将不会做任何事情，也不会产生错误。
  • index_name：要删除的索引的名称。
  • CASCADE | RESTRICT：这两个关键字用于处理依赖对象。CASCADE会删除索引以及依赖于该索引的所有对象（例如，依赖于该索引的外键约束）。RESTRICT是默认行为，它会在存在任何依赖对象时阻止删除索引。

    注意：DELETE语句是DML（数据操作语言）语句，它只删除数据，而不释放表的空间。因此，频繁使用DELETE可能会导致表膨胀，并需要定期进行VACUUM操作来回收空间。

    注意：TRUNCATE语句会立即释放表的空间，并重置任何相关的序列。因此，在使用TRUNCATE之前，请确保了解它的这些行为。
